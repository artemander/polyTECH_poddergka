Техническая составляющая проекта. 
Телеграм-бот: 2 шт.
БД на 1С
Сервер APACHE для выгрузки 1С

Регистрация через Телеграм-бота происходит через интеграцию с 1С. При регистрации бот запрашивает входные данные в виде номера и ФИО. Эти данные проверяются на вхождение в одной из таблиц в 1С. После чего, при успешной проверке, данные о пользователе Телеграм попадают в таблицу с информацией, которая в затем используются для связи студентов/аспирантов/сотрудников, и их данные в дальнейшем используются при записи на занятия, и вывода списка предстоящих занятий. При большинстве команд, требующих регистрацию от пользователя идёт проверка, и предлагает пользователю зарегистрироваться. Большинство команд реализованы через HTTP-запросы к выгруженной 1С БД. Почти все требуют ключ регистрации при каждом подключении. Этот ключ указывается администратором при запуске Телеграм-бота. Сам ключ хранится в 1С, и проверяется на соответствие там же. На главной странице — выпадающий список "сервис". Там можно прочитать, изменить и просто просмотреть ключ, по которому будет подключатся внешний ресурс. В БД 1С хранится практически вся информация, используемая приложением. Исключением является расписание. Оно подключается через отдельный файл с расширением *.pkl. Этот файл считывается и изменяется при необходимости внутри самого Python, с помощью библиотеки pickle. Так же есть текстовый шаблон, который используется для формирования html страницы с расписанием. Он используется для подстановки значений из файла расписания, и формируется новая таблица на странице расписания. Менять расписание можно с помощь второго Телеграм-бота, но технология авторизации несколько отличается от предыдущей. Для это имя пользователя в Телеграм должно находится в таблице "Администраторы" внутри 1С. Это обязательное условие, иначе расписание не сможет изменять расписание. Регистрация внутри бота не представляется возможной. Можно просмотреть, на какие занятия записан тот или иной пользователь. Можно использовать Телеграм-бота, но будет показываться только информация одного пользователя. Внутри БД можно просмотреть все. Необходимые действия: загрузить БД внутрь 1С, выгрузить Информационную Базу на сервер APACHE. Обязательно должна располагаться в localhost. Также необходимо запустить ботов, у одного из них указать ключ подключения (запрашивается при запуске).
